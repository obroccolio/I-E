## 三、核心问题与应用场景

### 1. 核心问题

当前企业合同审核主要存在以下问题：

1. **审核效率低：** 常规合同人工审核通常需要 1 到 2 小时，复杂合同耗时更长，影响业务推进节奏。
2. **专业门槛高：** 非法务人员难以识别隐性风险、权责失衡条款和模糊表述，容易埋下后续纠纷隐患。
3. **审核标准不统一：** 不同人员对风险判断尺度不同，企业缺乏统一、可复用的合同审核标准。
4. **综合成本偏高：** 外部法务审核费用高，长期依赖人工处理会增加企业运营成本。

### 2. 主要应用场景

本项目适用于以下高频合同审核场景：

* 商务合作合同
* 劳动雇佣合同
* 房屋租赁合同
* 服务采购与外包合同
* 中小企业日常经营中的标准模板合同

## 四、MVP 功能规划

围绕课程周期与实现难度，小组拟将 MVP 聚焦在以下核心功能上：

| 功能模块 | 功能说明 | 优先级 |
| :--- | :--- | :--- |
| 合同上传与解析 | 支持 Word、PDF、图片等格式上传，完成文本提取与基础清洗 | P0 |
| 条款自动拆分 | 自动识别合同中的权责、违约、付款、期限等关键条款 | P0 |
| 风险扫描与分级 | 对显性风险、隐性风险和合规问题进行定位，并标注高/中/低风险等级 | P0 |

## 五、技术实现思路

本项目初步采用“文本解析 + 法律规则校验 + 大模型辅助生成”的技术路线：

1. **文本获取层：** 通过 OCR 与文档解析工具完成 PDF、Word、图片合同的文本提取。
2. **结构化处理层：** 对合同内容进行条款切分、字段抽取和重点条款定位，形成可分析的结构化结果。
3. **风险识别层：** 结合规则库与法律知识进行显性风险、歧义风险和合规问题识别。
4. **智能建议层：** 借助大模型生成修改建议、风险解释和报告摘要。
5. **结果输出层：** 生成可读的审核报告，便于课堂演示和实际业务使用。

从硬科技属性上看，项目的难点不在于简单关键词匹配，而在于：

* 法律文本的专业语义理解
* 非结构化合同内容的稳定解析
* 规则校验与模型生成结果的结合
* 风险提示结果的可解释性与可复核性

## 八、当前风险与应对思路

在项目初步规划阶段，已经可以预见以下几类风险：

1. **法律专业知识不足：** 当前团队对法律条文和典型风险条款的掌握还不够系统，需要通过资料学习和案例整理快速补足。
2. **数据样本有限：** 若缺乏足够真实合同样本，可能影响规则库覆盖范围，因此前期应优先收集公开模板与典型案例。
3. **模型输出不稳定：** 大模型在法律场景下可能出现表述不严谨或误判，需要通过规则校验和人工复核降低风险。
4. **时间投入有限：** 三位成员每周投入时间均约 8 小时，因此必须优先聚焦 MVP，不宜一开始追求过多扩展功能。

对应的基本应对策略是：先缩小 MVP 范围，优先保证核心链路可运行；先做高频合同类型，再逐步扩展；先以“规则 + 模型辅助”实现稳健结果，再考虑后续智能化增强。
